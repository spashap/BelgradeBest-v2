Platform Phase 1: claimable-listings engine + 12 Expo pavilion profiles

Implements Phase 1 of KB/platform/PLATFORM-PLAN.md (status updated in the
plan; CLAUDE.md documents the new machinery).

Listings engine:
- Per-listing data masters src/data/listings/<leg>/<slug>.json -> new
  src/lib/listings.ts: eager glob loader, validListing thin-content guard
  (summary >=80 chars, about >=300, facts >=2 — sub-threshold listings stay
  unpublished stubs), SECTION per-leg URL segment map (expo-2027 ->
  "pavilions"), newestUpdated for hub freshness.
- T4 listing template src/pages/expo-2027/pavilions/[slug].astro: at-a-glance
  facts table with per-row sources, announced/reported editorial paragraphs,
  quotes, sources list, "claim this page" note -> /for-businesses; JSON-LD
  Article + BreadcrumbList.
- Pavilion directory hub /expo-2027/pavilions: stat tiles, profiled list, and
  a directory row for every one of the 93 named participant countries
  (built-vs-pending, mirroring the leg-hub pattern); ItemList JSON-LD.
- noindex-first behind site-config.json -> programmatic.listingsIndexable
  (false): astro.config.mjs enumerates listing paths into NOINDEX + sitemap
  exclusion and mirrors the SECTION map; per-listing `updated` drives lastmod
  once the flag flips. Tracker cross-link gated on the same flag.

Content — 12 pavilion profiles, web-researched 2026-07-05, every fact
