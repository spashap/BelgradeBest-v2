Admin platform redesigned around the north star: booths, not pavilions

Restructures /admin/platform per the owner's core clarification: the atomic
unit of the platform is the BOOTH — the individual business inside a pavilion
(~10k prospects across the Expo; same pattern later for other legs).
Pavilions are just categories/containers; government pavilion pages are a
maybe-later bonus. The admin is now a drill-down PROSPECT DATABASE, not a
listings CMS. KB plan + CLAUDE.md context preserved; north star recorded.

New structure (drill-down):
- /admin/platform — LEGS as navigational cards (all 7 legs from the schema,
  ready to grow): businesses count headline + categories / with-email /
  contacted / claimed meta per leg; inactive legs render as ghost cards that
  open ready-to-set-up. Below: booth-level funnel (north-star numbers), GA4
  traffic w/ deltas, GSC indexation feedback.
- /admin/platform/[leg] — CATEGORIES as cards (Expo: pavilions), each showing
  its booth rollup (count / w-email / contacted / claimed) + page state.
  "Add category" form + one-click category creation for every named Expo
  country still missing one (from the tracker dataset).
- /admin/platform/[leg]/[category] — THE working screen: booth table designed
  for outreach sessions — add-business form (name/email/person/website/notes
  -> creates an unpublished prospect stub via new createListing()), per-row
  contact+notes+claimed save, stage badge with quick-advance buttons and
  auto-stamped dates, per-booth email draft (open mailto / copy), stage
  filter chips with counts, category-page controls tucked into the header.
- Old screens re-scoped as cross-cutting tools: /listings -> "Search" (flat
  filterable list), /outreach -> "Queue" (all-stage work queue; now shows
  parent context). Shared sub-nav: Legs · Queue · Search.

Machinery:
- platform-store: createListing() (slugify w/ diacritics strip, overwrite
  guard, minimal stub JSON — thin guard keeps it off the public site) used by
  the extended /api/admin/platform (action=create).
- New lib/admin/outreach-draft.ts: one draft builder for all screens;
  template resolution "<leg>:<type>" -> "<leg>" -> default; placeholders now
  include {parentName}/{parentUrl}.
- New "expo-2027:booth" template in outreach-templates.json: pitches the
  PARENT pavilion page and frames the booth's own page as reserved — a
  prospect stub's URL is 404 until claimed, so booth emails must never link
  it.
- AdminLayout: navigational card styles (.cards/.card + ghost) and crumbs.
